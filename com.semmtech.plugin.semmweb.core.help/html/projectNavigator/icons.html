Icon Usage

Project Navigator Icons

This table shows the icons used in the Project Navigator. Keep in mind that some of them could be used in more than one place. Anyway the meaning of each icon is always unique even if it is placed in different places.

Icons Name Description
Semantic Project A collection of above components including interrelationships, to be used by a modeller for overview and ease-of-accessibility. Furthermore is serves as a means for storing preferences and temporary caching of models relevant within the scope of the project.
Model Collection Virtual folder that contains the Models of the Project. The models are treated as a logical structure highlighting some specific aspects of it.
Workspace Model Denote a Model that lies into a Project of the workspace. This icon can be found either under the Model Collection or under the Imports of another Model. In the first case it can be expanded to see additional information: Imports and Namespaces
External Model Denote a Model that lies outside of the Workspace. That can be either a file in the HD or a Web resource. It is used only for the Imports
Import Collection Virtual folder that contains the all Imports used by Model. It also contains the Indirect Imports or rather the Imports of Imports
Cached Model Denote a local cache of an External Model. Caching a remote Model can bring great improvement to the Editor performance. Moreover make sure that a Model can be always loaded independently of its availability
Namespace Collection Virtual folder that contains all the Namespaces used by a Model
Namespace A Namespace URI are used to provide resources with a shared location

Project Navigator Icon Overlays

This table shows the overlays applied to the icon in the Project Navigator. An overlay is a small image applied over an icon. An overlay is used to add additional information about the state of an item.

Overlay Name Description
Download/Parse error Created in the bottom-left corner of a Model or an Import if they are malformed or impossible to reach
Indirect Import Created in the top-left corner of an Import if it is indirect
Under SVN Control Created in the bottom-right corner of any kind of file. Indicate that the marked file is under SVN Version control
